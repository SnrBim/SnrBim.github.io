ModelAudit is an automated tool for auditing Revit BIM models against project standards. It analyses the current document and all loaded linked Revit models, producing an Excel report with detailed data across 31 check categories.

---

## Interface

On launch a dialog opens with three sections.

### Configuration (Optional)

Path to an Excel configuration file. Controls which checks run, which Revit categories are exported, and which element parameters are included in the report.

**If the field is empty** — the built-in default configuration is used (heavy checks disabled, standard set of categories and parameters).

Buttons:
- **Browse…** — select an existing configuration file.
- **Generate…** — create a new configuration file with default values. A save dialog suggests the path `%APPDATA%\Sener\BimTools\ModelAudit\Config-{ModelName}.xlsx`. The file also includes an **Aliases** sheet with auto-generated short names for linked documents — these can be edited manually.

Below the field a **configuration summary** is displayed in three columns:
- left — number of active checks and categories (hover for the full list)
- centre — number of parameters per category
- right — configuration file name and time of last modification

If the configuration contains errors, an error message is shown instead of the summary.

> The config path is saved when **Run Audit** is clicked and restored on the next dialog open.

### Previous Report (Optional)

Path to a previous report. When provided, **Old** / **New** columns appear in the Summary sheet for value comparison between runs. Old cells contain hyperlinks to detail sheets of the previous report.

### New Report Path

Path for saving the new report. Generated by default as:

```
%APPDATA%\Sener\BimTools\Reports\ModelAudit\{yyyy-MM-dd HH-mm}\{ModelName}.xlsx
```

Date and time are part of the folder name — re-running within the same minute overwrites the file in the same folder.

Buttons:
- **Browse…** — select the path manually via a save dialog.
- **Default** — generate a new path with the current date and time.

> The path is saved when **Run Audit** is clicked.

### Running

- **Run Audit** (or **Enter**) — start the audit.
- **Cancel** (or **Esc**) — close the dialog without running.

Before starting, the tool checks whether the report file is accessible: if it is already open in Excel a Retry / Cancel dialog appears.

---

## Progress Bar

During analysis a progress window is shown with a text log of execution. The log is read-only. The **Stop** button interrupts processing.

---

## Report Structure

The report is an Excel file (`.xlsx`) with the following sheets:

### Summary

Summary sheet. Rows are checks, columns are documents (current + linked).

- Each row header contains the check name with a hyperlink to the detail sheet.
- Two columns per document: **Old** (value from the previous report) and **New** (current).
- Last column — **Duration** (check execution time, format mm:ss).
- Short document names in the header link to the `_Metadata` sheet.
- The sheet is frozen at two header rows and the first two columns.

Colour scheme:
- **Yellow background** — High status (value requires attention).
- **Grey / pale text** — Low status (value is normal or unchanged).

### Detail Sheets

One sheet per active check. Contains a table with data for all documents together. A horizontal border separates documents. Auto-filters are enabled. The first cell links back to Summary.

### \_Metadata

Table mapping short document names to full names. Short Name links return to Summary.

### \_Config\_\* (hidden)

Sheets `_Config_Checks`, `_Config_Categories`, `_Config_Parameters` store the configuration used during the run. They allow reproducing the analysis conditions from an archived report.

## Configuration Excel

Generated by the **Generate…** button. It is recommended to edit the file before running the audit.

### Checks Sheet

Controls check activity.

| Column | Description |
|--------|-------------|
| Check Name | Check name (matches code, do not change) |
| Active | `yes` / `no` (or `true` / `false` / `1` / `0`) |
| Sheet Name | Detail sheet name in the report |
| Description | Additional information |

### Categories Sheet

Controls active categories for FamInstances and FamTypes checks.

| Column | Description |
|--------|-------------|
| BuiltInCategory | Revit enum value, e.g. `OST_Walls` (do not change) |
| Active | `yes` / `no` (or `true` / `false` / `1` / `0`) |
| Category Name | Localised name (auto-filled, do not change) |

### Parameters Sheet

Controls exported element parameters.

- First row — headers: `Common`, then names of active categories.
- **Common** column — parameters exported for all active categories.
- Other columns — parameters specific to a category.
- Empty cells are ignored.
- A parameter is searched first in the instance, then in the type.
- Missing parameters are logged.

### Aliases Sheet

Short document names for Summary headers.

| Column | Description |
|--------|-------------|
| Full Name | Full document name (key, do not change) |
| Short Name | Short display name (editable) |

On generation, short names are created automatically by removing the common prefix and suffix from all document names. Documents absent from the Aliases file get their short names generated automatically at runtime.

## Tips and Notes

### Configuration

- **Empty ConfigPath** is equivalent to the built-in default — clearing the path is safe.
- The summary below the field updates immediately on path change — validation errors are visible before running.
- Validation runs again on Run. If the config is invalid, the run is blocked.
- **Unknown check name** in the Checks sheet → error listing valid names.
- **Unknown category** in Categories or Parameters sheet → error listing supported categories.
- Row order in Checks and Categories sheets does not matter.
- Configuration is re-read on every run (no caching).
- The configuration used during a run is stored in hidden sheets of the report.

### Document Names

- Short names appear in Summary headers — this keeps the table compact with many links.
- If a linked document appears after the config was generated, its name is generated automatically; add it manually to the Aliases sheet if needed.
- Edit the Aliases sheet in the config before running if auto-generated abbreviations are inconvenient.

### Report Paths

- The new report path is remembered between sessions. It is auto-generated on first launch.
- The **Default** button generates a new path with the current date — useful for re-running on a different day.
- Two runs within the same minute create a file in the same folder → the second run will overwrite it. Use **Browse** or **Default** to create a new path if both results must be kept.
- If the report file is open in Excel, a retry dialog appears. The audit will not start while the file is locked.
- The previous report path **is not filled automatically** — it must be set manually via Browse.

### Performance

- **FamInstances** and **FamTypes** checks are disabled by default — they iterate over all elements in the selected categories and extract parameters. Execution time depends on the number of active categories and model size.
- **FamSizes** check is disabled by default — for each family `EditFamily` + `SaveAs` to a temp folder is called. On large models this can take several hours.
- **WorksetElements** check is disabled by default — iterates over all elements in all worksets.

### Linked Models

- Only loaded Revit links are analysed.
- Unloaded links are not included in the analysis.
- Documents with the same `Title` are added only once.

### Report Data

- Numeric values (coordinates, areas, distances) are written as Excel numbers — this allows correct sorting and filtering.
- Composite values (e.g. a space-separated list of Ids) remain strings.
- Line breaks within parameter values are preserved.
